# 竞标方案生成器

## Overview

基于 bidding-strategy（竞标策略顾问）输出的策略建议书，结合用户补充的品牌资料和数据，**按章节逐步输出**完整的竞标方案内容。

核心原则：**策略先行，内容跟着策略走** —— 每一页内容都要服务于 Skill 1 确定的差异化定位和赢单策略。

## When to Use

- 已完成 bidding-strategy 的策略分析，拿到策略建议书
- 需要输出完整的竞标方案内容（用于填入PPT）

## 输入要求

| 输入项 | 来源 | 必须/可选 |
|--------|------|-----------|
| 策略建议书 | bidding-strategy 输出 | 必须 |
| 品牌 brief / 招标书 | 客户提供 | 必须 |
| 品牌基础资料 | 官网、抖音号、历史投放数据 | 必须 |
| 竞品数据 | 蝉妈妈/飞瓜/巨量算数等 | 建议提供 |
| 跑量素材案例 | 竞品+大盘+我方历史素材 | 建议提供 |
| 我方案例数据 | 同品类历史项目脱敏数据 | 建议提供 |
| 公司介绍/团队资料 | 公司素材库 | 可选 |

## 输出规则

1. **逐章输出**，用户确认/修改后再继续下一章
2. **语言风格**：专业但不空洞，数据+洞察+结论三段式
3. **重点章节**（策略建议书标注为"重点展开"的）输出到可直接填入PPT的颗粒度，包含小标题、要点、数据引用位置
4. **简写章节**输出框架要点即可
5. 需要用户补充实际数据的位置标注 `[需补充：xxx数据]`
6. 需要甲方配合的环节标注 `[需甲方配合：xxx]`

## 方案章节结构

### 第一章：封面 & 目录

- 项目名称、品牌名/logo、我方公司名、提案日期
- 章节导航目录
- 输出格式：标题文案 + 目录结构

### 第二章：项目理解

**核心目标**：证明"我懂你"，展示洞察力。

- 对品牌需求的精准复述 —— 不是照搬brief，而是**比客户自己说得更清晰**
- 提炼核心目标、挑战、机会点
- 用一页纸讲清"我们理解的项目全貌"

**写法要求**：
- 先列客户原始需求要点，再用"我们的理解是……"进行升华
- 点出客户brief里没写但实际存在的潜在挑战
- 结尾用一句话总结项目的核心命题

### 第三章：市场与竞品分析

**核心目标**：用数据建立策略说服力。

| 子模块 | 内容 | 数据来源 |
|--------|------|---------|
| 行业趋势 | 品类大盘走势、增长驱动力、渠道格局变化 | 行业知识库附件 + 用户补充 |
| 品牌现状诊断 | 品牌在各渠道的声量、内容表现、投放效率 | 用户提供品牌数据 |
| 竞品打法拆解 | 竞品在各渠道的投放策略、内容方向、达人矩阵 | 蝉妈妈/飞瓜等 |
| 目标人群洞察 | 人群画像、行为路径、决策因素 | 巨量算数/品牌资料 |

**写法要求**：
- **数据说话，结论先行** —— 每组数据必须导向一个策略推导
- 格式：结论 → 数据支撑 → 策略启示
- 避免堆砌数据不下结论

### 第四章：内容策略（独立重点章节）

**核心目标**：展示内容能力和方法论深度，这是核心竞争力展示。

| 子模块 | 内容 | 分析逻辑 |
|--------|------|---------|
| 竞品素材拆解 | 竞品在各渠道的跑量素材分析：类型、结构、hook、卖点排列、时长分布 | 找到竞品内容的共性和可超越的空间 |
| 大盘跑量素材分析 | 同品类/同价格带的高消耗素材共性：开头形式、内容结构、转化组件 | 提炼品类内容的"基本面" |
| 我方成功素材案例 | 内部历史爆款素材复盘：结构拆解、成功归因 | 证明我们有可复用的方法论 |
| 可复用资产提炼 | 从以上分析中提炼可迁移的：脚本结构、拍摄手法、卖点话术、视觉风格 | 从分析到方法论的转化 |
| 内容方向规划 | 本项目的内容方向矩阵（N个方向 × 对应人群 × 预期效果） | 落地为可执行的内容计划 |
| 素材产能规划 | 月产量、类型配比、AB测试机制、迭代节奏 | 展示可持续的内容供给能力 |

**写法要求**：
- 核心逻辑链：**数据拆解 → 成功归因 → 方法论提炼 → 执行规划**
- 素材拆解需要有具体的结构化框架（如：前3秒hook类型、卖点排序、转化引导方式）
- 成功归因要做人群×内容×投放的交叉分析，不能只归因于单一变量
- `[需补充：竞品跑量素材截图/数据]` `[需补充：我方历史爆款素材数据]`

### 第五章：全案策略

**核心目标**：展示策略的系统性和跨渠道整合能力。

根据项目涉及的渠道和类型，动态组合以下子模块：

**5.1 整体策略框架**
- 一页纸讲清全案策略逻辑：目标→人群→渠道→内容→转化
- 各渠道的角色定位（种草/放量/收割/品宣）

**5.2 投放策略**（按渠道拆分）

根据客户涉及的渠道，逐一输出。**所有渠道使用统一分析维度**，便于横向对比和预算决策：

**统一分析维度：**

| 分析维度 | 说明 |
|---------|------|
| 渠道角色定位 | 该渠道在整体策略中承担的角色（拉新/种草/收割/品宣） |
| 人群定向策略 | 核心人群圈选逻辑、分层定向方案 |
| 出价与预算策略 | 出价模式、日预算分配、成本控制目标 |
| 素材/内容策略 | 该渠道适配的素材类型、内容方向、数量规划 |
| 转化链路设计 | 用户从广告触达到最终转化的完整路径 |
| 核心考核指标 | 该渠道的KPI定义和目标值 |
| 平台特殊机制 | 该渠道独有的流量规则、工具、红利点 |

**涉及渠道**（根据项目按需输出）：千川、巨量AD、CID引流（京东京准通/天猫阿里妈妈）、腾讯广告、快手磁力金牛、小红书聚光

每个渠道按以上7个维度逐一填写，确保可横向对比。

**5.3 达人策略**（如涉及）
- 达人矩阵规划（头部/腰部/尾部/素人配比）
- 达人筛选标准和合作模式
- 种草内容方向与品牌调性对齐

**5.4 跨渠道协同**
- 用户触达路径设计（如：小红书种草→抖音短视频放量→CID引流至京东/天猫成交）
- 各渠道数据打通与归因方案
- 预算在渠道间的动态调配机制

**5.5 直播协同策略**（如涉及）
- 短视频投放与直播的节奏配合
- 直播场次的流量承接策略
- 直播前蓄水/直播中引流/直播后追投的完整链路

### 第六章：执行方案

| 子模块 | 内容 |
|--------|------|
| 整体排期 | 对齐大促节点（618/双11/年货节等）和品牌节点，按阶段拆分 |
| 阶段划分 | 测试期→放量期→稳定期→大促冲刺期，各阶段目标和策略差异 |
| 素材产出计划 | 月产量、类型配比、拍摄/剪辑排期、审核流程 |
| 投放执行节奏 | 日/周/月的投放动作、预算分配、调优频次 |
| 数据监测 | 监测指标体系、数据看板、异常预警机制 |
| 优化机制 | 周报/月报节奏、优化决策流程、AB测试计划 |
| 阶段复盘 | 复盘节点、复盘维度、策略迭代机制 |

### 第七章：我方优势

| 子模块 | 内容 |
|--------|------|
| 核心团队 | 项目负责人、关键操盘手简历及成功案例 |
| 同品类案例 | 同品类/同量级成功案例，带脱敏数据截图 |
| 资源壁垒 | 达人资源、平台关系、工具/系统、媒体授权 |
| 服务模式 | 团队配置、沟通机制、响应时效 |
| 续约率/客户证言 | 用数据证明服务质量（如适用） |

### 第八章：预算与KPI

| 子模块 | 内容 |
|--------|------|
| 预算分配 | 按渠道/按阶段/按模块的预算分配表 |
| KPI体系 | 过程指标（曝光、点击、互动）+ 结果指标（GMV、ROI、CPA） |
| 数据口径 | 各渠道数据来源、归因规则、核算方式说明 |
| 费比说明 | 服务费占比及价值论证，对标行业水平 |
| 对赌/激励机制 | 阶梯式KPI、达标奖励/未达惩罚（如适用） |

### 第九章：服务保障 & 风险预案

| 子模块 | 内容 |
|--------|------|
| 日常服务SOP | 沟通频次、汇报节奏、需求响应时效 |
| 风险场景 | 达人违约/塌房、平台政策变动、成本波动、舆情危机、大促流量竞争 |
| 应对预案 | 每个风险场景对应的具体应对措施和备选方案 |
| 合规承诺 | 广告合规、数据安全、品牌调性保护 |

### 第十章：甲方资源配合需求

**核心目标**：体现专业落地思维，向客户明确方案执行所需的资源配合，管理预期。

| 配合维度 | 具体需求 | 对应方案环节 |
|---------|---------|-------------|
| 素材资源 | 产品图/视频素材、模特/场景授权、历史素材开放 | 内容策略、素材产出 |
| 达人授权 | 达人合作审批流程、调性要求、禁用红线 | 达人策略 |
| 工厂/溯源 | 工厂拍摄许可、原料产地探访安排 | 内容方向（溯源类） |
| 库存备货 | 大促期间供应链保障、爆款备货深度 | 投放放量节奏 |
| 价格机制 | 促销价格审批、优惠券/赠品权限、控价规则 | 转化策略、利益点设计 |
| 品牌资产 | 代言人肖像授权、IP使用许可、品牌VI规范 | 内容策略、素材制作 |
| 直播排期 | 品牌自播计划、大场直播档期、主播合作安排 | 短视频×直播协同 |
| 数据权限 | 店铺后台数据、历史投放数据、人群包共享 | 市场分析、投放优化 |
| 审批流程 | 素材审核时效承诺、修改轮次上限、紧急通道 | 全流程效率保障 |

**写法要求**：
- 语气专业但不强硬，定位为"为了保障效果的专业建议"
- 每项配合需求说明**为什么需要**（关联到方案中的具体策略）
- 标注优先级：必须项 / 强烈建议 / 可选项
- 给出时间节点：哪些需要合作前确认，哪些可以执行中逐步对齐

### 第十一章：结语

- 合作愿景（1-2段，结合项目特点和双方优势）
- 联系方式
- 下一步行动建议

## 章节权重动态调整

根据策略建议书中的**方案侧重建议**，各章节输出深度不同：

| 权重 | 输出标准 |
|------|---------|
| **重点展开** | 完整内容，含小标题、要点、数据引用位、示例，可直接填入PPT |
| **标准** | 框架完整，要点清晰，用户补充数据后即可使用 |
| **简写** | 提纲式输出，列出关键点和结构 |

通常的权重分配参考：
- 重点展开：内容策略、全案策略、预算与KPI
- 标准：项目理解、市场竞品分析、执行方案、甲方配合需求
- 简写：封面目录、我方优势（有模板可复用）、服务保障、结语
