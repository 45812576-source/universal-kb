# 客户成功管理（Customer Success Management）

## 概述

帮助 Martech/广告代理业务的客户成功经理系统化管理客户全生命周期，从签约到续约/扩展。

**核心原则：** 客户成功不是客服——客服是被动响应问题，客户成功是主动创造价值。你的工作是确保客户实现他购买时期望的业务成果（Desired Outcome）。

**中国市场特殊性：** 在中国，纯卖工具很难让客户持续付费。客户成功的本质是**"交付驱动续费"**——通过可见的服务交付物（报告、策略、优化动作）让客户感知到价值，而非仅靠产品使用数据。

**与销售策略顾问的关系：** sales-strategy-advisor 覆盖签约前的销售流程；本技能覆盖签约后的客户成功管理。两者在"续约/扩展"阶段有交集。

## 使用方式

**六种使用场景：**

1. **客户健康诊断：** 输入客户数据/行为信号 → 输出健康评分、风险等级、干预建议
2. **需求挖掘与翻译：** 输入客户反馈/对话 → 输出深层需求分析、产研需求文档
3. **流失预警与挽回：** 输入流失信号 → 输出预警等级、挽回策略、沟通话术
4. **续约/扩展策略：** 输入客户档案 → 输出续约策略、扩展机会、价值回顾材料
5. **沟通节奏规划：** 输入客户组合 → 输出分层沟通计划、触点设计
6. **服务交付规划：** 输入客户类型/阶段 → 输出交付物清单、价值呈现方案

## 理论基础（方法论来源）

本技能整合了以下主流客户成功方法论，并针对中国 Martech 市场做了本土化适配：

|-------|------|---------|--------------|

## 核心思维框架：客户成功飞轮（LAER+中国适配）

```
签约(Land) → 启动(Onboard) → 激活(Adopt) → 价值实现(Value) → 扩展(Expand) → 续约(Renew)
  ↑                                                                                    |
  └────────────────────────────────────────────────────────────────────────────────────────┘
```

**每个阶段的核心任务：**

| 阶段 | 核心任务 | 成功标准 | 失败信号 | 关键交付物 |
|-----|---------|---------|---------|-----------|
| **签约→启动** | 销售交接、设定预期、搭建服务框架 | 客户清楚接下来会发生什么 | 交接断层、客户不知道找谁 | 项目启动会PPT、服务SOP |
| **启动→激活** | 让客户快速上手，跑通核心流程 | 关键功能/流程跑通，首次数据回传成功 | 启动拖延、对接人不配合 | 配置手册、培训记录 |
| **激活→价值实现** | 让客户看到可量化的业务成果 | KPI达标或趋势向好，客户主动认可 | KPI未达标、客户质疑效果 | 周报/月报、优化建议 |
| **价值实现→扩展** | 发现新需求，增加合作范围 | 客户主动提需求、愿意增购 | 客户无新需求、预算收紧 | 增长方案、ROI分析 |
| **扩展→续约** | 确保客户续约，最好增购 | 提前续约、金额增长 | 拖延续约、要求降价 | 年度价值总结报告 |

**中国市场关键差异：**

```
西方模式：产品使用 → 价值实现 → 续约
中国模式：服务交付 → 可见交付物 → 关系信任 → 续约
```

在中国，即使产品数据表现好，如果客户"感知不到"你在做事，续约依然有风险。反之，即使数据一般，如果客户感受到你在积极服务、持续优化，续约概率反而更高。

## 模块1：客户健康诊断

**核心目标：** 通过多维度指标评估客户当前状态和未来走向，提前发现风险、识别机会。

### 1.1 DEAR+R 健康评分模型（Gainsight DEAR + 中国关系维度）

| 维度 | 权重 | 评估内容 | 绿灯 | 黄灯 | 红灯 |
|-----|------|---------|------|------|------|
| **D-部署(Deployment)** | 15% | 产品是否部署到位、功能是否开通 | 全部功能已开通并配置完成 | 部分功能未配置或未使用 | 核心功能未部署或配置错误 |
| **E-互动(Engagement)** | 20% | 客户与我方的沟通频率和质量 | 定期沟通、主动提需求 | 沟通频率下降、只被动回应 | 联系不上、不回消息 |
| **A-使用(Adoption)** | 20% | 产品实际使用深度和广度 | 核心功能日常使用、多人使用 | 只用基础功能、使用频率下降 | 几乎不登录、只有1人偶尔用 |
| **R-成果(ROI)** | 25% | 客户是否实现了预期的业务成果 | KPI达标或趋势向好 | KPI未达标但有改善趋势 | KPI远未达标、客户质疑效果 |
| **R-关系(Relationship)** | 20% | 关键人关系深度、多线程程度 | 多层级多部门关系、高层互访 | 只有单一对接人 | 关键人离职/换人、关系断裂 |

**评分规则：** 每个维度 0-100 分，加权计算总分。

| 总分区间 | 健康等级 | 含义 | 行动 |
|---------|---------|------|------|
| 80-100 | 🟢 健康 | 客户状态良好，有扩展机会 | 寻找扩展机会，深化关系 |
| 60-79 | 🟡 关注 | 存在风险信号，需要主动干预 | 制定干预计划，增加触点 |
| 40-59 | 🟠 预警 | 多个维度出现问题，流失风险高 | 启动挽回Playbook，升级处理 |
| 0-39 | 🔴 危险 | 客户即将流失或已表达不满 | 紧急干预，高层介入 |

### 1.2 Success Vector：看趋势而非快照

**Lincoln Murphy 的核心洞察：** 静态健康评分是滞后指标，真正重要的是**方向和速度**。

| 状态 | 含义 | 行动优先级 |
|-----|------|-----------|
| **↗ 上升** | 各维度持续改善，客户越来越活跃 | 低——保持节奏，寻找扩展机会 |
| **→ 平稳** | 指标稳定但没有增长 | 中——警惕，平稳≠健康，可能是停滞 |
| **↘ 下降** | 指标在恶化，即使当前分数还不低 | 高——立即干预，下降趋势比低分更危险 |
| **👻 沉默** | 关键联系人停止互动 | 最高——沉默是最危险的信号 |

**关键原则：** 一个当前80分但持续下降的客户，比一个当前60分但稳步上升的客户更危险。

### 1.3 中国市场特有的健康信号

**正面信号：**
- 客户主动在微信群里提需求或问问题
- 客户邀请你参加他们的内部会议
- 客户主动介绍其他部门/同事认识
- 客户在朋友圈转发你的内容
- 客户愿意做案例/接受采访

**危险信号：**
- 微信消息已读不回，或回复越来越慢
- 对接人说"我们内部先讨论一下"然后没有下文
- 开始问竞品的信息
- 月度/季度会议频繁取消或推迟
- 付款开始拖延
- 关键决策人换人且新人态度冷淡
- 客户开始要求导出所有数据

```

基本信息：
- 客户名称：[XX]
```

## 模块2：需求挖掘与翻译

**核心目标：** 从客户的表面反馈中挖掘真实需求，并翻译成产研团队能理解和执行的语言。

### 2.1 Desired Outcome 框架（Lincoln Murphy）

**客户成功 = 客户实现期望成果（Desired Outcome）**

```
期望成果 = 必要成果(Required Outcome) + 适当体验(Appropriate Experience)
```

- **必要成果**：客户必须达成的目标（"投放ROI达到1.2"）——这是硬指标
- **适当体验**：客户希望以什么方式达成（"不用我操心，你们帮我搞定"）——这是软需求

**关键洞察：** 竞品也能帮客户达成必要成果，但适当体验才是你的差异化。

### 2.2 三层需求挖掘法

| 层级 | 问什么 | 目的 | 示例 |
|-----|--------|------|------|
| **表层：现象** | "您目前遇到什么问题？" | 了解客户描述的症状 | "投放效果不好" |
| **中层：原因** | "这个问题是什么时候开始的？之前试过什么方法？" | 理解问题的上下文和历史 | "换了新品后ROI就下降了，试过调人群没用" |
| **深层：动机** | "如果这个问题解决了，对您意味着什么？如果不解决呢？" | 挖掘真正的驱动力和恐惧 | "老板给了3个月时间，做不起来这条线就砍掉" |

**SPIN问题框架（用于深度挖掘）：**

- **S-情境**："你们现在投放主要在哪些渠道？团队多少人？"
- **P-问题**："目前最让你头疼的问题是什么？"
- **I-暗示**："这个问题如果持续下去，会有什么影响？"
- **N-需求效益**："如果能解决这个问题，理想状态是什么样的？"

### 2.3 需求 vs 要求：关键区分

**刘徽（AfterShip VP）的核心洞察：** "反馈的是客户的需求，而不是客户的要求。"

| | 客户的要求（Demand） | 客户的需求（Need） |
|---|---|---|
| 定义 | 客户说"我要一个5mm的钻头" | 客户其实需要"一个5mm的孔" |
| 特点 | 具体的解决方案 | 底层的问题/目标 |
| 风险 | 照做可能做错方向 | 理解后可能有更好的方案 |
| 处理 | 追问"为什么需要这个" | 翻译成产品/服务可以解决的问题 |

### 2.4 需求翻译：从客户语言到产研语言

**客户说的 → 你要翻译成的：**

| 客户原话 | 表面含义 | 深层需求 | 产研语言 |
|---------|---------|---------|---------|
| "能不能加个XX功能" | 想要某个功能 | 某个场景下的效率问题 | 场景：[XX]；问题：[XX]；影响：[XX] |
| "你们的报表不好用" | 对报表不满意 | 看不到他关心的数据维度 | 用户角色[XX]需要在[场景]下看到[数据]以便[决策] |
| "竞品有这个功能" | 竞品对比压力 | 担心落后或有具体场景需要 | 需验证：是真实场景需求还是竞品焦虑 |
| "太复杂了" | 操作困难 | 学习成本高或流程不符合习惯 | 用户在[步骤]卡住，期望[XX]步完成 |

### 2.5 需求文档模板（给产研团队）

```
需求翻译文档

来源客户：[客户名称] — [客户类型] — [合同金额]
提出时间：[XX]
紧急程度：[高/中/低]
影响范围：[仅此客户 / 多个客户有类似需求 / 行业通用需求]

一、场景描述
- 谁：[什么角色的人]
- 在什么时候：[什么触发时机]
- 想做什么：[具体操作]
- 目前怎么做：[现有的替代方案/workaround]
- 痛苦是什么：[效率低/出错/看不到数据/...]

二、客户原话（保留原始表达）
"[客户的原话]"

三、需求分析
- 表面要求：[客户说要什么]
- 真实需求：[客户其实需要什么]
- 判断依据：[为什么这么判断]

四、建议方案（CS视角）
- 方案A：[XX] — 优点[XX] 缺点[XX]
- 方案B：[XX] — 优点[XX] 缺点[XX]

五、商业影响
- 不做的风险：[客户可能流失 / 影响续约 / 无影响]
- 做了的价值：[提升续约率 / 可复用给其他客户 / 差异化竞争力]
- 涉及客户数量：[XX个客户有类似需求]
```

## 模块3：流失预警与挽回

**核心目标：** 建立系统化的流失预警机制，在客户决定离开之前发现信号并干预。

### 3.1 流失分类矩阵（Lincoln Murphy）

| | **可避免** | **不可避免** |
|---|---|---|
| **可预期** | ⚠️ 需要反思——预警系统发现了但干预失败。分析干预是否太晚或方法不对 | ✅ 最好的坏结果——客户不匹配、倒闭、被收购。不应计入CSM考核 |
| **不可预期** | 🔴 最坏情况——好客户突然流失，说明预警系统有盲区。必须系统性复盘 | 🔍 需要调查——突发事件导致。检查预警系统是否能更早发现 |

**核心规则：** 如果客户还在经营、还在付其他供应商的钱、而且当初是合适的客户——那这个流失一定是可避免的。

### 3.2 预警信号分级

**🔴 红色预警（需48小时内响应）：**
- 客户明确表达不满或要求终止合作
- 关键决策人离职或换人
- 客户开始接触竞品并索要对比方案
- 连续2周以上联系不上对接人
- 客户要求导出全部数据

**🟠 橙色预警（需1周内响应）：**
- 产品使用量连续2周下降超过30%
- 月度/季度会议被取消或推迟
- 客户对服务质量提出投诉
- KPI连续未达标且客户开始质疑
- 付款开始拖延

**🟡 黄色预警（需2周内关注）：**
- 对接人回复速度明显变慢
- 客户不再主动提需求或反馈
- 只使用基础功能，不愿尝试新功能
- 客户内部组织架构调整
- 行业/市场环境发生重大变化

### 3.3 挽回Playbook（按预警等级）

**红色预警 Playbook：**

```
第1步（当天）：内部对齐
- 召集CS经理+销售+产品，同步情况
- 梳理客户完整历史：合同、沟通记录、KPI数据、异议记录
- 判断流失原因类别：效果不达标/关系断裂/竞品替代/预算削减

第2步（48小时内）：高层介入
- 安排CS总监或VP直接联系客户决策人
- 不要急于解释或承诺，先倾听
- 话术："我听说您对合作有一些顾虑，我想亲自了解一下情况"

第3步（1周内）：制定挽回方案
- 根据流失原因制定针对性方案
- 效果不达标 → 提供免费优化期+专项优化方案
- 关系断裂 → 更换CSM+高层定期互访
- 竞品替代 → 差异化价值分析+迁移成本提醒
- 预算削减 → 降级方案+核心价值保留

第4步（2周内）：执行并跟踪
- 每3天跟进一次进展
- 确保承诺的改善措施落地
- 如果挽回成功，制定防复发计划
```

**沉默流失专项 Playbook（最危险的流失类型）：**

```
信号：客户不投诉、不提需求、不回消息，但也不续约

第1步：换渠道触达
- 微信不回 → 打电话
- 电话不接 → 发短信/邮件
- 都不回 → 找其他联系人

第2步：价值先行，不谈续约
- 不要上来就问"您还续约吗"
- 话术："最近整理了一份您行业的投放趋势报告，想分享给您"
- 或："我们产品最近上了一个新功能，觉得对您特别有用"

第3步：正式升级
- 如果2周内仍无回应，安排高层正式拜访
- 话术："我们非常重视和贵司的合作，想当面聊聊接下来的合作方向"

第4步：接受现实
- 如果所有努力都无效，做好交接准备
- 保持专业和友好，不要撕破脸
- 留好口碑，客户可能会回来
```

## 模块4：续约与扩展策略

**核心目标：** 在合同到期前系统化推进续约，同时识别和把握扩展机会。

### 4.1 续约时间线（倒推法）

| 时间节点 | 行动 | 负责人 |
|---------|------|--------|
| **续约前120天** | 内部评估客户健康度，判断续约风险等级 | CSM |
| **续约前90天** | 启动价值回顾，准备年度价值总结报告 | CSM |
| **续约前60天** | 与客户进行价值回顾会议，试探续约意向 | CSM + CS经理 |
| **续约前45天** | 提出续约方案（含扩展建议），开始商务谈判 | CSM + 销售 |
| **续约前30天** | 解决所有遗留问题，推进合同签署 | 销售 + 法务 |
| **续约前14天** | 如未签约，升级处理 | CS总监 |
| **续约当天** | 如仍未签约，启动紧急挽回流程 | 高层 |

### 4.2 价值回顾会议（中国版QBR）

**会议频率：**

| 客户等级 | 月度服务回顾 | 季度战略回顾 | 年度价值总结 |
|---------|------------|------------|------------|
| S级（战略客户） | 每月 | 每季度 | 每年（高层参加） |
| A级（重点客户） | 每月 | 每半年 | 每年 |
| B级（标准客户） | 每2月 | 无 | 每年（简版） |
| C级（自助客户） | 无（自动报告） | 无 | 续约前1次 |

**季度战略回顾（QBR）结构：**

```
1. 回顾本季度成果（15分钟）
   - KPI达成情况（用数据说话）
   - 完成的关键动作
   - 对比上季度的改善

2. 价值量化（10分钟）
   - ROI计算
   - 效率提升数据
   - 成本节约数据
   - 与行业基准对比

3. 问题与挑战（10分钟）
   - 遇到的问题及解决方案
   - 仍待解决的问题
   - 需要客户配合的事项

4. 下季度规划（10分钟）
   - 目标设定
   - 策略调整
   - 新功能/服务预告

5. 扩展机会探讨（5分钟）
   - 基于数据发现的增长机会
   - 新产品/服务推荐（自然带出，不硬推）
```

**关键原则：** QBR不是汇报会，是价值确认会。目标是让客户说出"确实有效果"。

### 4.3 扩展机会识别

**扩展信号：**
- 客户使用量接近或超过当前套餐上限
- 客户主动问"你们还有什么其他产品/服务"
- 客户的业务在增长（新品牌、新渠道、新市场）
- 客户内部其他部门开始关注你的产品
- 客户愿意做案例、参加活动（说明认可度高）

**扩展策略：**

| 扩展类型 | 触发条件 | 策略 |
|---------|---------|------|
| **增量扩展** | 使用量增长、新增渠道 | "您的投放量增长很快，升级到XX套餐可以获得更好的数据支持" |
| **交叉销售** | 客户有其他产品能解决的需求 | "您提到素材效率的问题，我们的XX产品正好能解决" |
| **深度服务** | 客户需要更多人工服务 | "基于您的需求，建议升级到专业版服务，包含XX" |
| **新部门拓展** | 其他部门有类似需求 | "您的XX部门是不是也在做投放？我们可以一起聊聊" |

### 4.4 续约谈判中的常见挑战（中国市场）

| 挑战 | 应对策略 |
|-----|---------|
| **客户要求降价** | 先做价值回顾，用数据证明ROI；如果必须让价，要换取承诺（增加合作范围、延长合同期） |
| **客户拿竞品比价** | 不要陷入价格战，强调差异化价值和迁移成本；"换供应商的隐性成本您考虑过吗？" |
| **关键人换了** | 尽快与新人建立关系，重新做一次价值回顾；同时维护与其他联系人的关系 |
| **客户说"再考虑考虑"** | 追问具体顾虑，不要等；"方便告诉我您在考虑什么吗？我看能不能帮您解决" |
| **客户要求免费延期** | 可以给短期延期（1-2周），但要换取明确的决策时间；不要无限延期 |

## 模块5：沟通节奏规划

**核心目标：** 根据客户分级和生命周期阶段，设计系统化的沟通节奏，确保不遗漏、不过度。

### 5.1 客户分级标准

| 等级 | 标准 | 服务模式 | CSM配比 |
|-----|------|---------|---------|
| **S级** | 年合同额>50万 或 战略标杆客户 | 专属CSM + 专属项目经理 | 1:3-5 |
| **A级** | 年合同额10-50万 | 专属CSM | 1:8-15 |
| **B级** | 年合同额3-10万 | 共享CSM | 1:20-30 |
| **C级** | 年合同额<3万 | 自助+社群 | 1:50+ |

### 5.2 沟通节奏矩阵

| 触点类型 | S级 | A级 | B级 | C级 |
|---------|-----|-----|-----|-----|
| **数据日报** | 每日自动推送 | 每日自动推送 | 每周汇总 | 自助查看 |
| **微信轻触达** | 每周2-3次 | 每周1-2次 | 每2周1次 | 无 |
| **电话/视频沟通** | 每周1次 | 每2周1次 | 每月1次 | 无 |
| **月度服务回顾** | 每月（正式PPT） | 每月（简版） | 每2月 | 无 |
| **季度战略回顾** | 每季度（高层参加） | 每半年 | 无 | 无 |
| **年度价值总结** | 每年（高层互访） | 每年 | 每年（简版） | 续约前1次 |
| **节日问候** | 每个重要节日 | 春节+中秋 | 春节 | 群发 |
| **行业资讯分享** | 每周 | 每2周 | 每月 | 社群统一 |

### 5.3 微信沟通规范

**微信是中国B2B客户成功的核心渠道，必须认真对待：**

| 场景 | 响应时间要求 | 注意事项 |
|-----|------------|---------|
| 客户提问/求助 | 工作时间30分钟内 | 即使不能立即解决，也要先回复"收到，我马上看" |
| 客户投诉 | 15分钟内 | 先安抚情绪，再解决问题 |
| 日常分享/闲聊 | 2小时内 | 保持友好互动，不要已读不回 |
| 非工作时间消息 | 次日上班后30分钟内 | 如果是紧急问题，当晚回复 |

**微信沟通技巧：**
- 主动分享有价值的信息（行业报告、竞品动态、平台政策变化）
- 适当的非工作话题互动（节日祝福、朋友圈点赞评论）
- 重要事项用文字确认，不要只口头说
- 建立客户专属微信群（包含CSM+项目经理+技术支持）

### 5.4 铁三角协作模式（华为模型适配）

**大客户建议采用铁三角团队：**

| 角色 | 职责 | 与客户的关系 |
|-----|------|------------|
| **客户经理（AM）** | 负责商务关系、续约谈判、扩展销售 | 对接客户决策层 |
| **解决方案经理（SM）** | 负责方案设计、产品咨询、技术支持 | 对接客户业务/技术层 |
| **交付经理（DM）** | 负责日常服务交付、数据报告、运营优化 | 对接客户执行层 |

**协作规则：**
- 每周铁三角内部同步会（15分钟）
- 客户信息统一记录在CRM/客户档案中
- 任何一方发现风险信号，立即通知其他两方
- 对外口径一致，不要让客户感觉"你们内部没沟通"

## 模块6：服务交付与价值呈现（中国市场核心）

**核心目标：** 通过系统化的服务交付，让客户"看得见、摸得着"你在创造的价值。

### 6.1 交付驱动续费模型

```
中国市场续费公式：
续费意愿 = 业务成果 × 服务感知 × 关系信任

三个因子缺一不可：
- 业务成果好 + 服务感知差 + 关系一般 = 可能续费但会砍价
- 业务成果差 + 服务感知好 + 关系好 = 可能给机会但有期限
- 业务成果好 + 服务感知好 + 关系好 = 大概率续费且可能增购
```

### 6.2 各阶段交付物清单

| 阶段 | 交付物 | 频率 | 目的 |
|-----|--------|------|------|
| **启动期** | 项目启动会PPT | 1次 | 设定预期、建立信任 |
| | 配置完成确认书 | 1次 | 确认交付标准 |
| | 操作培训记录 | 1-2次 | 确保客户会用 |
| **运营期** | 数据日报/周报 | 每日/每周 | 让客户看到你在做事 |
| | 优化建议报告 | 每2周 | 展示专业价值 |
| | 月度服务回顾PPT | 每月 | 正式价值确认 |
| | 竞品动态简报 | 每月 | 额外价值输出 |
| **续约期** | 年度价值总结报告 | 1次 | 量化ROI，支撑续约 |
| | 下年度策略建议 | 1次 | 展示前瞻性 |

### 6.3 数据报告设计原则

**报告不是给自己看的，是给客户的决策者看的：**

| 原则 | 说明 | 示例 |
|-----|------|------|
| **结论先行** | 第一页就是核心结论和关键数字 | "本月ROI 1.3，环比提升15%" |
| **对比说话** | 有对比才有感知 | 环比、同比、vs目标、vs行业均值 |
| **可视化** | 图表比数字更直观 | 趋势图、对比柱状图、占比饼图 |
| **行动导向** | 每个数据后面跟"所以我们要做什么" | "CPC上升10%，建议优化素材方向" |
| **品牌化** | 报告要有你的品牌标识和专业排版 | 统一模板、公司logo、专业配色 |

### 6.4 价值量化框架

**帮客户算清楚你值多少钱：**

| 价值维度 | 量化方式 | 话术 |
|---------|---------|------|
| **效果提升** | ROI提升幅度、GMV增长 | "使用我们的系统后，ROI从0.8提升到1.2，提升了50%" |
| **效率提升** | 节省的人力时间 | "跨平台数据自动汇总，每天节省2小时手动对数据的时间" |
| **成本节约** | 减少的浪费 | "异常消耗预警帮您避免了XX万的无效消耗" |
| **风险规避** | 避免的损失 | "链路监控在XX次故障中及时预警，避免了XX万损失" |
| **决策支持** | 数据驱动的优化 | "基于数据分析，我们调整了渠道分配，整体ROI提升了XX%" |

### 6.5 标杆客户打造策略

**投资打造2-3个标杆客户，用他们的成功故事驱动获客和留存：**

```
标杆客户选择标准：
- 行业代表性强（大健康/美妆头部品牌）
- 合作效果好（有可量化的成果数据）
- 客户关系好（愿意配合做案例）
- 品牌知名度高（名字本身就是背书）

标杆打造投入：
- 提供高于标准的服务水平
- 安排最优秀的CSM和项目团队
- 定期高层互访
- 优先获得新功能/新服务

标杆客户产出：
- 联合案例（PDF/视频/公众号文章）
- 客户证言（文字/视频）
- 行业活动联合演讲
- 新客户参访接待
```

## 核心指标体系

### 北极星指标

| 指标 | 计算方式 | 目标 |
|-----|---------|------|
| **净收入留存率（NRR）** | (期初ARR + 扩展 - 流失 - 缩减) / 期初ARR | >100%（优秀>110%） |
| **毛收入留存率（GRR）** | (期初ARR - 流失 - 缩减) / 期初ARR | >85%（优秀>90%） |

### 过程指标

| 指标 | 含义 | 追踪频率 |
|-----|------|---------|
| 客户健康分布 | 绿/黄/橙/红各占比 | 每周 |
| 首次价值实现时间（TTFV） | 签约到客户首次认可效果的天数 | 每客户 |
| QBR完成率 | 按计划完成的QBR占比 | 每月 |
| 预警响应率 | 预警信号在规定时间内响应的占比 | 每周 |
| 扩展率 | 产生扩展收入的客户占比 | 每季度 |
| NPS/满意度 | 客户推荐意愿 | 每半年 |

### 中国市场特有指标

| 指标 | 含义 | 为什么重要 |
|-----|------|-----------|
| 微信响应速度 | 客户消息平均响应时间 | 中国客户把响应速度等同于重视程度 |
| 交付物按时率 | 报告/方案按约定时间交付的占比 | 交付物是价值感知的核心载体 |
| 关键人覆盖度 | 每个客户接触的决策链角色数 | 单线程关系是最大的流失风险 |
| 高层互访频次 | 双方高层见面的频率 | 高层关系是续约的安全网 |

## 常见错误

| 错误 | 正确做法 |
|-----|---------|
| **把客户成功当客服** — 只在客户找你时才响应 | 主动出击，定期触达，在客户发现问题之前发现问题 |
| **只看产品数据不看关系** — 觉得数据好就没问题 | 数据好+关系差=随时可能被替换；关系是中国市场的安全网 |
| **不做价值量化** — 觉得客户应该能感受到价值 | 客户感受不到的价值等于不存在；必须用数据和报告让价值可见 |
| **续约前才开始关注** — 平时不管，快到期了才着急 | 续约是全年工作的结果，不是最后一个月的冲刺 |
| **需求照单全收** — 客户要什么就做什么 | 区分需求和要求，翻译成产品语言，评估商业价值后再决定 |
| **单线程关系** — 只和一个对接人联系 | 必须建立多层级多部门的关系网，降低关键人风险 |
| **服务范围无边界** — 客户"顺便"提的都做 | 明确服务边界，超出范围的要沟通并适当收费 |
| **不做流失复盘** — 客户走了就算了 | 每个流失客户都是学习机会，必须复盘并改进 |
| **过度承诺效果** — 为了留住客户什么都答应 | 站着赚钱，管理预期，承诺能做到的，做到承诺的 |
| **忽视沉默客户** — 不投诉就以为没问题 | 沉默是最危险的信号，主动触达沉默客户 |

## 客户成功十大法则速查（Nick Mehta）

1. **卖给对的客户** — 不匹配的客户注定流失，CS从获客就开始
2. **客户和供应商天然会疏远** — 不主动维护，关系必然衰减
3. **客户期望你让他们大获成功** — 不是"不出问题"，是"超出预期"
4. **持续监控和管理客户健康** — 不能测量就不能管理
5. **个人关系不再足以建立忠诚** — 关系+价值才能留住客户
6. **产品是唯一可规模化的差异化** — 服务建立关系，产品建立壁垒
7. **执着地缩短价值实现时间** — 越快看到效果，留存率越高
8. **深度理解客户的指标** — 用客户的语言衡量成功，不是你的
9. **用硬指标驱动客户成功** — CS必须数据化，不能靠感觉
10. **这是自上而下的全公司承诺** — CS不是一个部门的事
