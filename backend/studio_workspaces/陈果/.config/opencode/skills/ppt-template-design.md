# PPT Template Analysis & Guide Generation

## Overview

PPT 模板不是配色方案——它是一套完整的设计体系。这个 Skill 教你如何从任何 .pptx 模板中提取设计体系，并输出一份可执行的模板指引文件，供 `pptx-generation` Skill 在生成 PPT 时直接引用。

## When to Use

- 用户提供了公司 PPT 模板文件（.pptx）
- 需要分析模板的设计语言并生成指引
- 需要确保生成的 PPT 与品牌模板视觉一致

## 模板分析方法论（5步）

### Step A: 解构模板文件

1. 解压 .pptx（本质是 zip），检查 `ppt/slides/` 下的 slide XML
2. 检查 `ppt/slideMasters/` 和 `ppt/slideLayouts/` 了解母版和布局
3. 逐页分析每张 slide 的：
   - 背景处理（纯色 / 渐变 / 图片）
   - 装饰元素（条、线、水印、图标）
   - 文字规格（字体、字号、字重、颜色、位置）
   - 布局坐标（精确到 EMU，转换为 Inches）
4. 检查 `ppt/media/` 目录，提取所有嵌入的图片资源（logo、装饰图等）
5. 识别 logo/品牌标识：
   - 在哪些页面出现？（封面、每页、结尾？）
   - 精确位置和尺寸（x, y, w, h）
   - 是图片引用还是形状组合？
   - 如果是图片，记录文件名（如 `image1.png`），供生成时 `add_picture()` 使用

```python
# 解压模板分析示例
import zipfile, os
with zipfile.ZipFile('template.pptx', 'r') as z:
    z.extractall('template_extracted')
# 然后用 Read 工具逐个检查 ppt/slides/slide*.xml
```

### Step B: 提取设计体系

从 XML 数据中归纳以下维度：

#### B1: 基础设计体系

- **配色系统**：主色、强调色、辅助色、语义色（警告/成功等）+ 透明度体系
- **字体层级**：封面→章节→页标题→卡片标题→正文→注释，每级的字体/字号/字重/颜色
- **布局网格**：边距、标题位置、内容区起始、页脚位置、logo位置
- **装饰元素库**：强调条、水印、渐变、分隔线、图标风格等

#### B2: 卡片/容器精确参数提取

对每种卡片类型，必须提取：
- 外框尺寸（宽×高）和圆角值（adj 值）
- 内边距（上/右/下/左，精确到 0.01"）
- 顶部色条高度（如有）
- 文字框定位方式（零内部padding + 精确坐标 vs 内部padding）
- 文字锚定方式（top/middle/bottom）
- autofit 设置（shrink/none/resize shape）
- 行距（spcPct 值，如 125%）
- 卡片内元素垂直间距（图标→标题→正文）
- 特殊构造（如编号圆圈的外圈/内圈直径、透明度、偏移量）

**输出格式**：每种卡片一张参数表，包含可直接用于代码的数值。

#### B3: 布局安全区（防止重叠）

必须定义每种页面类型的安全区：
- **标题区**：y 起始 → y 结束（含字号占用高度）
- **内容区**：y 起始 → y 结束
- **页脚区**：y 起始 → 页底
- **各区之间的最小间距**

**强制规则**：内容区 y 起始 = 标题区 y 结束 + 间距，绝不允许重叠。输出时必须给出每种页面类型的安全区数值表。

#### B4: 颜色使用比例规则

分析模板时必须统计每种颜色的实际使用场景和面积比例，输出：

| 颜色 | 使用场景 | 面积占比 | 禁止场景 |
|------|---------|---------|---------|
| 例：白色 | 正文、卡片背景 | 大面积 | — |
| 例：强调紫 | 色条、边框、圆圈、卡片标题 | ≤15% | 深色背景上的大面积正文 |

**强制规则**：
- 深色背景上的大面积正文必须用白色或浅色（高对比度）
- 强调色（紫/青等）只用于小面积元素：标题、边框、色条、圆圈
- 如果强调色与背景色同色系（如紫色在深蓝上），禁止用于正文
- 每页强调色面积不超过总面积的 15%
- 正文透明度、页脚透明度、装饰透明度必须分别记录

#### B5: Logo/品牌标识提取

必须提取：
- Logo 图片文件（从 `ppt/media/` 中识别）
- 出现位置规则（每页固定位置 / 仅封面结尾 / 仅封面）
- 精确坐标和尺寸（x, y, w, h，Inches）
- 是否有多个版本（彩色/白色/单色）

**输出**：指引文件中必须包含 logo 的 `add_picture()` 代码模式，以及在哪些页面类型上使用。

### Step C: 布局多样性清单（逐页提取）

对模板中的**每一页**，记录：

1. **背景处理方式**：纯色 / 分割（上下或左右）/ 图片叠加 / 渐变
2. **是否使用装饰图片**：位置、透明度、是否出画（部分超出页面边界）
3. **内容容器类型**：无容器 / 白卡 / 边框卡 / 渐变卡 / 半透明卡
4. **布局结构**：单列 / 双列 / 三列 / 四列 / 自由布局
5. **布局分类与提取**（分两大类，提取深度不同）：

   **A类：容器布局**（卡片网格、分栏、等宽列）
   → 已由 Step B2 覆盖（卡片参数表），此处只需记录布局结构（几列、间距、对齐方式）

   **B类：图解式布局**（时间轴、辐射图、大圆排列、数据可视化、不对称网格、层叠卡片、分屏布局、条带水印等——任何非卡片网格的布局）
   → 必须为每种图解式布局输出「构造配方」：
   - 所有形状的类型（OVAL / ROUNDED_RECTANGLE / FREEFORM 等）、尺寸、位置(x,y)、颜色、透明度
   - 形状之间的空间关系（间距、对齐方式、连接线角度/粗细）
   - 文字框的位置（相对于形状的偏移规则）
   - 可直接转化为 python-pptx 代码的参数表

然后统计：
- 模板共使用了几种不同的布局结构？
- 其中 A类（容器布局）几种？B类（图解式布局）几种？
- 装饰图片出现在多少页？位置模式是什么？
- 哪些布局是模板独有的（不是通用套路）？

**布局丰富度自检**（强制）：
- 模板中的图解式布局（B类）有几种？（如果 ≥ 3种，指引文件必须全部包含构造配方）
- 容器布局的变体有几种？（白底边框 / 彩色填充 / 半透明填充 / 底部色条 / 层叠 等）
- 如果模板有 N 种布局（A类+B类），指引文件中的「页面类型配方」必须覆盖至少 N-2 种
- 如果自检发现指引文件遗漏了图解式布局，必须补充后才能输出

**输出时必须包含「图解式布局配方」章节**（升级自原「模板特色布局」），包括：
- 每种图解式布局的完整构造配方（参数表+代码骨架）
- 独特的容器构造方式（如双层圆圈编号）
- 独特的背景处理（如大面积商业图片部分出画）
- 独特的元素组合（如色条+圆角卡片+透明度层次）

### Step D: 分析视觉节奏

- 页面之间的背景交替模式（全色 vs 分割 vs 图文）
- 内容密度波动（密集 → 呼吸 → 密集）
- 卡片/容器样式的轮换规则
- 强调色的交替使用

#### 构图重心规则

- 每页内容的视觉重心应居中或略偏上
- 结束语/总结语应与主体内容对齐（不要孤立在角落）
- 如果主体内容是左对齐列表，结束语也应左对齐或居中
- 底部留白用于页脚，不要塞内容
- 多列卡片布局时，卡片组整体应水平居中于内容区

#### Presentation 文字层次规则

指引文件必须指导生成代码如何制造文字的视觉层次感，而非平铺直叙：

- **拆分制造戏剧感**：一句话如果有转折/递进，应拆成两个文本框——前半句小字平述，后半句大字跳出（如 "今天不讲理论" 28pt → "只做一件事" 52pt）
- **关键词 highlight**：正文中的核心概念用强调色+bold 突出，其余保持正文色。但 highlight 必须克制——每页不超过2-3个关键词
- **字号跳跃要够大**：相邻层级字号差至少 8-12pt，差距太小会显得层次不清（如标签18pt → 标题36pt → 核心56pt）
- **颜色层级配合字号**：最大的字用最亮的色（白色），中等字用次亮色（浅灰），最小的字用最暗色（暗灰）。强调色只用于需要"跳出来"的关键词，不用于整段文字

### Step E: 输出模板指引文件

将以上分析编译为一份结构化的指引文件（见下方"模板指引文件规范"）。

#### 创造性变化原则

指引文件必须包含「变化手法库」章节：

**1. 同类型页面的变化手法**：
- 同样是3列卡片，可以变化：卡片高度、有无图标、边框色、有无底部色条
- 同样是编号列表，可以变化：水平排列 vs 垂直排列、圆圈大小、有无描述文字
- 同样是内容页，可以变化：有无装饰图片、背景是否分割

**2. 跨页面的节奏变化**：
- 不要连续3页以上使用相同的内容区起始 y 坐标
- 每4-5页至少有一次布局结构的显著变化
- 装饰元素（椭圆、图片、水印）应分散使用，不要集中或完全不用

**3. 禁止清单**：
- 禁止连续2页使用完全相同的页面类型
- 禁止全文档只用1种卡片样式
- 禁止所有内容页使用相同的标题位置+内容区起始组合
- 禁止装饰元素在全文档中完全缺席

#### 生成代码质量规则（踩坑经验）

指引文件必须包含以下技术约束，供 `pptx-generation` 生成代码时遵守：

**颜色严格约束**：
- 生成代码中只允许使用模板配色表中列出的颜色，不得自创颜色
- 每种颜色有明确的使用场景（如 TEXT_DIM 只用于页脚/时间标注，不用于标题标签）
- 指引文件必须为每种颜色标注「允许场景」和「禁止场景」

**文本框尺寸适配**：
- 文本框高度应贴合字号：单行文本框高度 ≈ 字号(pt) × 0.018(inch) + 0.1" 余量
- 避免文本框过高导致文字"漂浮"在大空间中
- 多行文本用 `a:lnSpc > a:spcPct` 设置行距（如 125000 = 125%），不要用 `space_after`
- 含 `\n` 换行的文本框必须设置 `line_spacing_pct`，否则行距不可控

**容器内文字适配**：
- 圆圈/椭圆内放文字时，必须验证文字宽度 < 圆圈直径 × 0.7
- 如果文字装不下，优先缩短文字（如 "01" → "1"），其次放大容器，最后缩小字号
- 卡片内文字行数 × 行高 不得超过卡片内容区高度

## 模板指引文件规范

每份指引文件必须包含以下章节：

```
# [模板名称] 模板指引

## 1. 设计体系
### 1.1 配色系统（hex + 角色 + 透明度变体）
### 1.2 字体层级（层级 → 字体/字号/字重/颜色）
### 1.3 布局常量（边距、标题位置、内容区、页脚等，精确到 Inches）

## 2. 通用元素
每页内容页必须包含的固定元素及其代码实现

## 3. 页面类型配方
每种页面类型的：
- 适用场景
- 背景处理
- 布局结构（精确坐标）
- 装饰元素
- 代码模式（python-pptx）

**必须包含的页面类型**：
- 至少2种容器布局变体（如：边框卡3列、实心卡2列、半透明卡2×2）
- 模板中所有图解式布局（如：时间轴、辐射图、大圆排列、分屏、不对称网格等）
- 每种图解式布局必须附带完整的 python-pptx 代码模式（不只是参数，要有可执行的函数）

## 4. 容器/卡片系统
所有卡片样式的代码实现（边框卡、填充卡、渐变卡等）

## 5. 装饰元素库
所有装饰元素的代码实现

## 6. 视觉节奏规则
页面排列的节奏指导

## 7. 颜色使用比例规则
每种颜色的使用场景、面积占比、禁止场景表格
深色背景正文用色规则、强调色面积上限

## 8. 卡片精确参数表
每种卡片类型的完整参数：尺寸、圆角、内边距、色条、
文字框定位、锚定、autofit、行距、元素间距

## 9. 布局安全区定义
每种页面类型的标题区/内容区/页脚区 y 坐标范围
各区最小间距，防重叠规则

## 10. 图解式布局配方
对模板中每种图解式布局（非卡片网格的布局），必须输出：

### [布局名称]（如：同心圆辐射图）
- **适用场景**：什么内容适合用这种布局（如"核心概念+3个辐射子概念"）
- **构造参数表**：
  | 元素 | 形状类型 | 尺寸(w×h) | 位置(x,y) | 颜色 | 透明度 |
  |------|---------|-----------|-----------|------|--------|
  | 外圈 | OVAL | 4.58"×4.58" | (4.38, 1.50) | #7153EE | 100% |
  | 中圈 | OVAL | 2.97"×2.97" | (5.18, 2.30) | #BBAEF8 | 79% |
  | ... | ... | ... | ... | ... | ... |
- **空间关系**：形状之间的间距、对齐、连接线规则
- **文字位置**：相对于形状的偏移规则
- **python-pptx 代码骨架**：可直接复制使用的函数

（从 Step C 图解式布局构造配方中提炼，每种布局必须有完整的参数表和代码骨架）

## 11. 变化手法库
同类型页面的变化维度
跨页面节奏变化规则
禁止清单

## 12. Logo/品牌标识
Logo 图片文件路径
出现规则（哪些页面类型需要 logo）
精确位置和尺寸
add_picture() 代码模式

## 13. 生成代码质量约束
颜色严格约束（只允许使用的颜色清单+场景）
文本框尺寸适配规则
容器内文字适配规则
文字层次感规则（拆分、highlight、字号跳跃）
```

## 内容类型 → 页面类型映射指南

通用的内容-视觉映射原则（不限于特定模板）：

| 内容特征 | 推荐视觉处理 |
|---------|------------|
| 开场/结尾 | 居中、大字、装饰性强、呼应 |
| 章节切换 | 极简、大留白、编号+标题 |
| 并列观点(2-4个) | 等宽卡片网格 |
| 递进/步骤 | 编号+连接元素(箭头/线) |
| 对比(A vs B) | 双列、差异化配色 |
| 关键洞察/金句 | 大字居中、装饰引号、呼吸页 |
| 叙事/案例 | 图文分割或信息分层释放 |
| 详细数据 | 白底卡片、深色文字 |
| 流程/步骤(3-6步) | 垂直时间轴 + 双层圆点 |
| 核心概念+辐射子概念 | 同心圆辐射图 |
| 并列能力/工具(3-4个) | 彩色大圆水平排列 |
| 关键数据/百分比 | 大数字视觉焦点 或 圆形数据可视化 |
| 1个核心+多个子项 | 不对称网格(1大+N小) |
| 图文结合的叙事 | 左图右文分屏 |
| 层次递进关系 | 多层叠加卡片 |
| 目录/章节概览 | 深色条带+大水印文字 |

**图解式布局优先规则**：如果模板提供了图解式布局（时间轴、辐射图、大圆排列等），在内容特征匹配时**必须优先使用图解式布局**，而非退回到通用的卡片网格。卡片网格是兜底方案，不是默认方案。

## 视觉节奏通用规则

1. **连续密集内容不超过3页** → 插入呼吸页（引用/过渡/大字）
2. **背景处理至少2种交替**（全色 vs 分割 vs 图文）
3. **容器样式不连续重复**（边框卡 → 填充卡 → 渐变卡）
4. **每个逻辑章节**：过渡 → 内容 → 呼吸
5. **强调色交替使用**，避免单一强调色疲劳
