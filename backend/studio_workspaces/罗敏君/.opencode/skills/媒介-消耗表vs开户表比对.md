# 消耗表 vs 开户表 比对

## 任务说明
比对"消耗表"和"开户表"两个 Excel 文件，找出消耗表中存在但开户表中缺失的记录，输出缺失明细到一个新的 Excel 文件。

## 执行前确认
- 运行前必须先搜索桌面上的消耗表和开户表文件，列出所有匹配项，通过 AskUserQuestion 让用户确认使用哪个文件
- 若 $ARGUMENTS 中明确指定了文件路径，则无需确认

## Sheet 匹配规则（按 sheet 名一一对应）

| Sheet | 消耗表匹配列 | 开户表匹配列 | 备用匹配列 | 辅助输出列（来自消耗表） |
|-------|------------|------------|-----------|---------------------|
| 字节 | 广告主账户id | 广告主ID | - | 简称, 广告主账户名称, 客户名称 |
| 京准通 | LoginSubPin | PIN | - | 简称, 广告主公司名称, 代理子Pin ID |
| 康淘 | 账户id | 账户id | - | 简称, 账户名称 |
| UDS | 媒体投放账户id | UD-ID | 巨量ID | 简称, 店铺名称, 媒体投放账户, 媒体 |

## 匹配逻辑
1. 两张表的 sheet 名称一一对应：字节对字节、京准通对京准通、康淘对康淘、UDS 对 UDS
2. **列名必须精确匹配**（忽略大小写），不使用模糊匹配，防止匹配错误
3. 取消耗表中匹配列的所有唯一值，逐一去开户表对应匹配列中查找
4. 匹配时统一转为字符串并去除首尾空格，做精确比对
5. 消耗表中存在但开户表中找不到的，即为"开户表缺失"项
6. 缺失项去重（同一个 ID 只输出一次）
7. UDS 特殊逻辑：先用消耗表的 `媒体投放账户id` 匹配开户表的 `UD-ID`；若匹配不上，再用同一个 `媒体投放账户id` 匹配开户表的 `巨量ID`；两轮都匹配不上才算缺失
8. 终端输出 UDS 时需分别显示 UD-ID 匹配数、巨量ID 补匹配数、缺失数

## 异常处理
1. **缺少 sheet**：若消耗表或开户表中缺少某个平台 sheet（如没有"康淘"），跳过该 sheet 并在终端提示，不影响其他 sheet 的比对
2. **匹配列空值**：消耗表中匹配列为空值的行，单独标记并输出到结果中（标注为"空值行"），不参与正常匹配逻辑

## 防护措施（防止匹配错误）
1. **精确匹配列名**：只使用精确匹配（忽略大小写），不使用包含匹配
2. **显示匹配结果**：运行时显示实际匹配到的列名，如 `"广告主账户id" -> "广告主账户id"`
3. **ID格式校验**：字节、康淘、UDS 的 ID 必须是纯数字格式，如果匹配到的值不是数字（如"委托"、"字节"等文本），会报错终止
4. **错误终止**：如果有任何校验失败，不会输出错误结果，而是提示用户检查配置

## 输出
- 在桌面生成 `开户表缺失比对结果-{日期}.xlsx`，每个 sheet 对应一个比对结果
- 每个 sheet 包含：匹配列的值 + 辅助输出列的信息
- 在终端打印各 sheet 的汇总统计（消耗表唯一ID数、开户表唯一ID数、缺失数）
