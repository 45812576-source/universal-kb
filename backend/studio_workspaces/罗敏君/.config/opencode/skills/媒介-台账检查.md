# 媒介-台账检查

## 功能概述

对字节、京准通、智汇投三大媒体的流水表、开户表、台账进行清洗、匹配和比对，验证台账（人工登记）是否与系统流水一致，定位差异客户。

**支持平台：**
- **字节**：有量端口 + 蜂窠端口
- **京准通**：天机有量端口 + 天机智投端口
- **智汇投**：天机有量端口（腾讯/小红书/B站/字节/快手/预算通多平台）

---

## 使用方法

### Node.js 脚本

```bash
# 字节台账检查
node 媒介-台账检查.js 字节

# 京准通台账检查
node 媒介-台账检查.js 京准通

# 智汇投台账检查
node 媒介-台账检查.js 智汇投
```

### Web 端

打开 `媒介-台账检查.html`，通过顶部 Tab 切换平台。

## 通用处理流程

### 第一步：数据整合
- **字节/京准通**：合并两份流水表（有量+蜂窠/智投+有量）
- **智汇投**：清洗流水，删除"支付宝充值"行

### 第二步：客户匹配

| 平台 | 匹配字段 | 匹配逻辑 |
|-----|---------|---------|
| 字节 | 广告主ID | 转入方/转出方账户ID → 开户表广告主ID → 客户简称 |
| 京准通 | PIN | 转入/转出账户去掉括号 → PIN → 客户名称 |
| 智汇投 | shopname/旺旺号 | 备注中提取 → shopname/旺旺号 → 客户名称 |

### 第三步：端口分类

| 平台 | 端口识别 | 计算方式 |
|-----|---------|---------|
| 字节 | 转入方一代账户名称含"有量"/"蜂窠" | 充值-退款 |
| 京准通 | 端口主体列（天机有量/天机智投） | 充值-退款 |
| 智汇投 | 固定天机有量端口 | 支出-存入 |

### 第四步：生成备款表
- 按日期+端口分组计算
- 字段：日期、端口、客户提单时间、当日退款总额、当日充值使用金额、差值

### 第五步：台账清洗
筛选条件：
1. 日期匹配（支持多日期范围）
2. 媒体平台限定（各平台对应媒体）
3. 排除纯打款、抵扣、返点
4. 端口匹配

### 第六步：核对比对
- **偏差 = 备款差值 − 台账合计**
- **一致性判断（满足任一即视为一致）：**
  - 偏差绝对值 ≤ 0.01
  - 偏差绝对值 / 备款差值绝对值 < 0.001%（适用于大金额多笔交易场景）
- 支持多日期拆分显示

### 第七步：客户级差异定位（如有差异）
1. 流水按客户汇总各端口金额
2. 台账按客户汇总折算账户币
3. **客户名称模糊匹配**（前缀匹配，如"渝冬来"="渝冬来-千川"）
4. 输出差异客户、金额、偏差、来源（仅流水/仅台账/双方）

## 输出结果

### 文件输出
- 桌面生成 `[平台]台账检查结果.xlsx`
- 包含 sheet：
  - `[平台]流水(清洗后)`
  - `[平台]开户表`
  - `[平台]台账`
  - `备款`
  - `总体比对`
  - `客户差异明细`

### 控制台输出
- 各步骤处理统计
- 按日期比对结论
- 差异客户明细（如有）

## 技术要求
- 使用 Node.js + xlsx 模块
- xlsx 路径：`C:/Users/Administrator/node_modules/xlsx/xlsx.js`
- 金额计算保留两位小数（`Math.round(x * 100) / 100`）
- 匹配时统一转为字符串并 trim
- 折算账户币为文本格式，用 `parseFloat + replace(/,/g, '')` 解析
- 智汇投CSV需自动检测编码（优先尝试UTF-8，失败则回退GBK解码）
