# 媒介-销售政策查询

## 功能概述

查询各平台返点政策，方便商务随时查询，减少重复咨询。

内置 2026 年最新政策数据，开箱即用。

---

## 数据结构

政策数据使用数组存储，每条政策包含以下字段：

```javascript
{
  platform: "平台名称",      // 如：字节、阿里妈妈、京准通、二代-腾讯
  category: "业务类型",      // 如：竞价广告、千川、腾讯、快手
  license: "牌照/政策名称",  // 如：竞价八类、千川二代、腾讯-微盟-K1框
  scope: "适用范围",         // 如：引流电商非大健康、客保内、本地
  policy: "政策内容",        // 如：对私最高4.5%，对公最高1.5%
  updateDate: "更新日期",    // 如：2026/3/12
  note: "备注说明"           // 特殊说明、政策变动提示
}
```

---

## 模糊查询规则

### 1. 查询逻辑

用户输入关键词后，应对以下所有字段进行**不区分大小写的模糊匹配**：
- `platform` - 平台名称
- `category` - 业务类型
- `license` - 牌照名称
- `scope` - 适用范围
- `policy` - 政策内容
- `note` - 备注说明

### 2. 查询示例

| 用户输入 | 应匹配到的政策 |
|---------|--------------|
| 腾讯 | 阿里妈妈-腾讯、二代-腾讯-微盟K1-K5框、腾讯-企量 |
| 千川 | 字节-千川二代、字节-千川直客 |
| 大健康 | 字节-竞价九类（医疗健康）、阿里妈妈-字节（大健康）、快手-浪险（大健康CID） |
| 浪险 | 二代-快手-浪险 |
| 4% | 所有包含"4%"的政策 |
| 品牌 | 所有在policy或note中提到"品牌"的政策 |

### 3. 结果展示

- 按平台分组展示
- 标注政策更新日期
- 如有备注（尤其是含"政策变动"），需高亮显示

## 伪代码实现

```
函数 searchPolicy(关键词):
  如果 关键词为空:
    返回 全部政策数据

  将关键词转换为小写
  结果 = 空数组

  遍历每条政策:
    如果 政策的任一字段（转小写后）包含关键词:
      将该政策加入结果

  返回 结果

函数 显示结果(结果列表):
  按 platform 字段分组
  每组内显示:
    - 牌照名称 (license)
    - 业务类型 (category)
    - 政策内容 (policy)
    - 适用范围 (scope，如有)
    - 更新日期 (updateDate)
    - 备注 (note，如有且含"政策变动"则高亮)
```
